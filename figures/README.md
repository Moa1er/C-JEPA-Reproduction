# Encoder validation figures

Outputs from the merged SAVi encoder pipeline run against CLEVRER
validation videos. Generated by `scripts/make_diagnostic_figures.py`,
which drives `src.encoders.savi_encoder.SAViEncoder` (the same wrapper
the slot-extraction pipeline uses) and feeds it clips preprocessed via
`src.data.transforms.build_frame_transform` against `configs/base.yaml`.

All three videos are mid-clip (`--start-frame 20 --stride 8`), each
showing a multi-object scene with motion. Forward passes are seeded
with `torch.manual_seed(42)` (SAVi is stochastic).

## Figure set

| Video | mp4 | CLI args | tracked slot | object |
| --- | --- | --- | --- | --- |
| v10800 | `data/clevrer_val/video_10000-11000/video_10800.mp4` | `--start-frame 20 --stride 8` | 6 | cyan sphere |
| v10500 | `data/clevrer_val/video_10000-11000/video_10500.mp4` | `--start-frame 20 --stride 8` | 1 | orange object |
| v10100 | `data/clevrer_val/video_10000-11000/video_10100.mp4` | `--start-frame 20 --stride 8` | 3 | red cylinder |

For each video, three PNGs are saved:

- `<tag>_slot_masks.png` — per-timestep overlay of each slot's mask on
  the input frame. Each slot should attend to a coherent region (object
  or background), not noise.
- `<tag>_slot_reconstructions.png` — ground truth, combined recon, and
  each slot's masked reconstruction in isolation. Per-slot reconstructions
  should look like masked versions of the original objects.
- `<tag>_temporal_consistency_slot<K>.png` — heatmap of slot K's mask
  across all 6 timesteps. A stable spatial extent over time indicates
  slot identity is preserved (the slot is tracking the same object).
